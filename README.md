# いとまき — オーダーメイド刺繍ECサイト

写真をアップロードすると自動で刺繍データに変換し、糸の色・太さ・サイズを調整して
Tシャツ・タオル・トートなどに配置 → そのまま購入できるECサイトのプロトタイプです。

## ひらきかた（3通り）

| 方法 | 手順 |
|---|---|
| ① いちばん簡単 | **index.html をダブルクリック**（Edge / Chrome で開く） |
| ② ローカルサーバ | このフォルダで `python -m http.server 4173` → http://localhost:4173 |
| ③ Node派 | `npx serve .` など任意の静的サーバでもOK |

ビルド・インストールは一切不要です。インターネット接続があると手書きフォント
（Google Fonts: Klee One / Yomogi / Zen Maru Gothic）が読み込まれます。
オフラインでも丸ゴシック系の代替フォントで動作します。

## あそびかた（動作確認コース）

1. トップ「さっそくつくる」→ 商品をえらぶ（例: Tシャツ）
2. エディタで **サンプル画像**（おはな/くまさん/さくらんぼ/にじ）をクリック
   → その場で刺繍プレビューに変換されます
3. 「刺繍のちょうせい」で色数・太さ・密度・角度・クロスステッチを触ってみる
   / 「つかう糸」をクリックすると48色から糸を差し替えられます
4. 「ばしょと大きさ」で位置プリセット・サイズ・かたむきを調整、
   仕上がりプレビュー上で**ドラッグ移動**
5. カートに入れる → ご注文手続き（デモ決済）→ 注文完了
6. フッター「注文管理（運営用）」で受注を確認し、
   **ステッチSVG / 仕様書JSON / プレビューPNG** をダウンロード

> アップロード画像はすべてブラウザ内で処理され、外部送信されません。
> カート・注文データはブラウザの localStorage に保存されます。

    ├── data/products.js  商品マスタ + 手描きSVGイラスト + 刺繍ゾーン定義
    ├── engine/embroidery.js  ★刺繍変換エンジン

    ├── ui/components.js  ヘッダー/フッター/モーダル/アイコン

## よくあるカスタマイズ

| やりたいこと | さわる場所 |
|---|---|
| 商品を追加・価格変更 | `js/data/products.js` の `IT.PRODUCTS`（+ 同ファイルの `ART` にイラスト） |
| 糸色を追加・変更 | `js/data/threads.js` |
| 刺繍代の計算式 | `js/store.js` の `embroideryFee()` |
| 送料・送料無料ライン | `js/store.js` の `cartTotals()`（520円 / 6,000円） |
| 特商法・お問い合わせ先 | `js/pages/staticPages.js` の `law`（［　］を自分の情報に） |
| ブランド名・ロゴ | `js/ui/components.js`（LOGO_SVG）と `index.html` の `<title>` |

## 刺繍データについて

- エディタ/注文の「刺繍データ(SVG)」は **1単位=1mm の実寸ステッチ座標**です。
  1本の線分 = 1針。色ごとにグループ化され、糸の品番・名前がメタデータに入っています。
- Inkscape + [Ink/Stitch](https://inkstitch.org/)（無料）に取り込めば、
  刺繍ミシン用データ（DST / PES など）へ変換する土台になります。
- 仕様書JSONには 商品・生地色・配置位置・針数・糸リスト（針数内訳つき）が入っており、
  外部の刺繍業者への発注書としても使えます。

## これは何ができて、何がモックか

**動くもの**: 画像→刺繍変換（減色/糸マッピング/背景除去/タタミ・クロス描画/針数計算）、
配置エディタ、価格リアルタイム見積、カート、注文フロー、注文管理、製作データ出力。

**モック**: 決済（課金されません）、メール通知、在庫、サーバ保存。
本番化の手順は `要件定義書.md` の「ロードマップ」参照
（Stripe等の決済 / Supabase等のDB / 独自ドメイン公開）。
