# いとまき — オーダーメイド刺繍ECサイト

写真をアップロードすると自動で刺繍データに変換し、糸の色・太さ・サイズを調整して
Tシャツ・タオル・トートなどに配置 → そのまま購入できるECサイトのプロトタイプです。

## ひらきかた（3通り）

| 方法 | 手順 |
|---|---|
| ① いちばん簡単 | **index.html をダブルクリック**（Edge / Chrome で開く） |
| ② ローカルサーバ | このフォルダで `python -m http.server 4173` → http://localhost:4173 |
| ③ Node派 | `npx serve .` など任意の静的サーバでもOK |

ビルド・インストールは一切不要です。インターネット接続があると手書きフォント
（Google Fonts: Klee One / Yomogi / Zen Maru Gothic）が読み込まれます。
オフラインでも丸ゴシック系の代替フォントで動作します。

## あそびかた（動作確認コース）

1. トップ「さっそくつくる」→ 商品をえらぶ（例: Tシャツ）
2. エディタで **サンプル画像**（おはな/くまさん/さくらんぼ/にじ）をクリック
   → その場で刺繍プレビューに変換されます
3. 「刺繍のちょうせい」で色数・太さ・密度・角度・クロスステッチを触ってみる
   / 「つかう糸」をクリックすると48色から糸を差し替えられます
4. 「ばしょと大きさ」で位置プリセット・サイズ・かたむきを調整、
   仕上がりプレビュー上で**ドラッグ移動**
5. カートに入れる → ご注文手続き（デモ決済）→ 注文完了
6. フッター「注文管理（運営用）」で受注を確認し、
   **ステッチSVG / 仕様書JSON / プレビューPNG** をダウンロード

> アップロード画像はすべてブラウザ内で処理され、外部送信されません。
> カート・注文データはブラウザの localStorage に保存されます。

## よくあるカスタマイズ

| やりたいこと | さわる場所 |
|---|---|
| 商品を追加・価格変更 | `js/data/products.js` の `IT.PRODUCTS`（+ 同ファイルの `ART` にイラスト） |
| 糸色を追加・変更 | `js/data/threads.js` |
| 刺繍代の計算式 | `js/store.js` の `embroideryFee()` |
| 送料・送料無料ライン | `js/store.js` の `cartTotals()`（520円 / 6,000円） |
| 特商法・お問い合わせ先 | `js/pages/staticPages.js` の `law`（［　］を自分の情報に） |
| ブランド名・ロゴ | `js/ui/components.js`（LOGO_SVG）と `index.html` の `<title>` |

## 刺繍データについて（ミシンでそのまま使えます）

注文ごとに、以下の**ミシン用データを直接ダウンロード**できます（管理画面/注文完了/マイページ）:

| 形式 | 用途 |
|---|---|
| **PES** | ブラザー **刺しゅうPRO / PE-Design で開いて編集可能**（CSewSegオブジェクト入りのv1形式）。家庭用ブラザーミシンでもそのまま縫えます |
| **DST** | タジマ形式。業務用ミシン・刺繍業者への入稿の業界標準 |
| SVG | 実寸(1単位=1mm)のステッチ座標。確認・アーカイブ用 |
| 仕様書JSON | 糸リスト（いとまき品番 + **ブラザー糸番号**）・実針数・トリム数・寸法 |

ミシンデータは**デジタイザーv2**（実務のプンチングと同じ「領域分類」方式）で生成されます:
- **輪郭ベクトル化**: マーチングスクエア法＋角保護スムージングで、解析セルの
  階段状ギザギザを除去したなめらかな輪郭を縫う
- **タタミ/サテン自動分類**: 幅約4mm以下の細い線・文字は**サテン縫い**
  （骨格線に沿ったジグザグ、距離変換で局所幅を追従）、広い面はタタミ縫い
  （サーペンタイン縫い順＋4位相レンガ針落ち）
- **縁取りラン**: タタミ面は仕上げに輪郭ランニングでエッジを立たせる
- **部位別の下打ち**: タタミ=エッジウォーク+直交走り縫い ／ サテン=センターウォーク(+ジグザグ)
- **止め縫い（タイイン/タイオフ）**: 各色ブロックとトリム前後に返し縫いでほどけ防止
- **引き縮み補正**: タタミ0.18mm・サテン0.12mmの張り出し
- **ノイズ処理**: 減色境界のにじみ粒は隣接する面へ吸収、糸幅より細いスライバーは省略、
  コンパクトな微小点（目など）は手差し数針で残す
- **縫い順最適化**: 近接順で走行を最小化、2.4mm超の渡りは自動トリム
- バイナリ仕様はリファレンス実装 [pyembroidery](https://github.com/EmbroidePy/pyembroidery) (MIT) 準拠で、
  同ライブラリでの読み戻し検証済み（`tools/verify-machine.mjs` + `tools/readback.py` で
  カバレッジ・はみ出し・針目長・寸法・決定性を自動検査）
- クロスステッチのみ従来方式（格子幾何そのもののため）。SVGダウンロードは
  「ミシンが縫う軌跡そのまま」（縫い順・下打ち込み）に変わりました

※ 刺しゅうPROでは「ステッチデータ」として開きます。オブジェクト単位の再編集をしたい場合は
　刺しゅうPRO側の「ステッチをブロックに変換」機能でブロック化してください。

## これは何ができて、何がモックか

**動くもの**: 画像→刺繍変換（減色/糸マッピング/背景除去/タタミ・クロス描画/針数計算）、
配置エディタ、価格リアルタイム見積、カート、注文フロー、注文管理、製作データ出力。

**動くもの（追加）**: 刺繍PRO用PES・タジマDSTのミシンデータ生成（下打ち・止め縫い・縫い順最適化つき）。

**モック**: 決済（課金されません）、メール通知、在庫、サーバ保存。
本番化の手順は `要件定義書.md` の「ロードマップ」参照
（Stripe等の決済 / Supabase等のDB / 独自ドメイン公開）。
