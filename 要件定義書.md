---

## 1. プロジェクト概要

### 1.1 コンセプト

> あなたの「すき」を、ちくちく刺繍に。

ユーザーがアップロードした写真・イラスト・ロゴを**自動で刺繍データに変換**し、
糸の色・太さ・密度・サイズを自分で調整して、好きな商品（Tシャツ・タオル・バッグなど）に
配置してそのまま購入できる、完全オーダーメイドの刺繍ECサイト。

### 1.2 ユーザー体験（3ステップ）

1. **えらぶ** — 商品（Tシャツ / タオル / トートなど）と生地の色を選ぶ
2. **つくる** — 写真をアップロード → 自動で刺繍プレビューに変換 → 糸色・太さ・サイズを微調整して配置
3. **とどく** — そのまま購入。実際に刺繍された商品が手元に届く

### 1.3 デザイン方針

- 手書き文字・手書きフォントをベースにした、**温かく・柔らかく・可愛らしく・おしゃれ**な世界観
- 生成り（きなり）の布のような背景、糸・針・スプールの手描きイラスト、ステッチ（破線）の飾り罫
- フォント: Yomogi / Klee One（手書き系）+ Zen Maru Gothic（丸ゴシック・本文用）
- 商品写真の代わりに**手描きタッチのSVGイラスト**を採用し、世界観を統一

#### (b) 刺繍データ自動変換（変換エンジン）
| 要件ID | 機能 | 内容 |
|---|---|---|
| CONV-1 | 減色 | OKLab知覚色空間でオーバークラスタリング(K+6)→知覚的マージにより指定色数（2〜12色）へ。面積が小さくても色の際立つ細部（目・口）がクラスタを持てる |
| CONV-2 | 糸色マッピング | 減色結果を**実在の刺繍糸パレット（48色）**へOKLab色差で自動割当 |
| CONV-3 | 背景除去 | 境界リングから背景色モデル(最大3つ)を推定→領域拡張＋**白フチ(にじみ)の浸食除去**＋文字の輪っか内など**囲まれた背景色の除去**。PNG透過も対応 |
| CONV-10 | 自動クロップ | 余白の大きい画像は被写体範囲を特定して切り抜き再解析（2パス）。実効解像度が上がり、サイズ指定＝モチーフ実寸になる |
| CONV-4 | ノイズ除去 | コントラスト保護つき3×3多数決フィルタ（色が大きく違う細部は保護） |
| CONV-8 | 線画強調 | 3×3サブピクセル解析で暗い線を検出し、輪郭・目・文字を保持（イラスト向け、ON/OFF可） |
| CONV-9 | ミシンデータ生成 | **PES（刺しゅうPRO/PE-Design対応・CSewSeg付きv1）/ DST（タジマ）** をブラウザ内で直接生成。縫い順最適化・止め縫い・下打ち・引き縮み補正・自動トリム・ジャンプ分割(≦12mm)・Brother 64色パレットへの糸番号割当を含む（pyembroidery準拠、読み戻し検証済み） |
| CONV-11 | デジタイザーv2 | ミシンデータは実務のプンチングと同じ**領域分類方式**で生成: (1)マーチングスクエア法＋角保護スムージングで輪郭をベクトル化（セル格子の階段を除去） (2)幅≦約4mmの細線・文字は**サテン縫い**（Zhang–Suen骨格＋距離変換で局所幅追従・センターウォーク下打ち） (3)広い面はタタミ（サーペンタイン縫い順・4位相レンガ針落ち・エッジウォーク＋直交下打ち・輪郭ランニング縁取り） (4)にじみ粒は隣接面へ吸収/糸幅未満のスライバーは省略/微小点は手差し。検証ハーネス（tools/verify-machine.mjs: カバレッジ≧96%・はみ出し≦0.5%・針目長≦7.2mm・寸法±1mm・決定性・72通りパラメータ掃引 / tools/readback.py: pyembroidery読み戻し）付き |
| CONV-5 | ステッチ描画 | タタミ縫い（並行ステッチ+レンガ状オフセット+糸の艶表現）を再現 |
| CONV-6 | クロスステッチ | ×印グリッドのクロスステッチモードに切替可能 |
| CONV-7 | 針数計算 | 生成ステッチの総長から針数を実計算（価格・納期に連動） |

## 5. 刺繍糸パレット

- 全48色。各糸に **和風の名前（さくら・よもぎ・るり 等）+ 品番（IT-01〜IT-48）+ 色値** を定義
- 減色結果は必ずこのパレットへマッピング（実際に調達可能な糸で作れることを保証する思想）

## 6. 料金体系

```
商品合計 = Σ（本体価格 + 刺繍代）× 数量
刺繍代   = 基本料 ¥600
         + 面積課金: 実刺繍面積(cm²) × ¥9（クロスステッチは ×¥7）
         + 色数加算: 4色目から 1色につき +¥120
         （10円未満切り上げ）
送料     = ¥520（¥6,000以上で無料） / 代引き手数料 +¥330
納期目安 = 5営業日 + 針数10,000針ごとに+1日
```

## 7. データモデル（プロトタイプ: localStorage）

```
itomaki_cart_v1   : CartItem[]
itomaki_orders_v1 : Order[]

CartItem {
  id, productId, colorId, apparelSize?, qty,
  design {
    params { style, colors, weight, density, angle, removeBg, bgTol },
    palette [{ threadId, count }],        // 使用糸と占有セル数
    labels { w, h, rle },                 // 減色ラベルマップ（RLE圧縮）
    srcThumb,                             // 元画像の小さなサムネイル(DataURL)
    stitchCount, coverage
  },
  placement { zoneId, dx, dy, widthMm, rot },
  price { base, embroidery, unit }
}

Order {
  id("IT" + 日付 + 連番), items, customer{...}, payment, totals{...},
  status: "新規受付" | "制作中" | "発送済み", createdAt
}
```

- 元画像そのものは保存しない（容量・プライバシー配慮）。製作に必要な情報は
  ラベルマップ+糸割当+パラメータで完結し、そこからステッチSVGを何度でも再生成できる。

## 8. 非機能要件

| 項目 | 要件 |
|---|---|
| 動作環境 | Chrome / Edge / Safari 最新版。**ビルド不要・サーバ不要**（index.htmlを開くだけで動作） |
| レスポンシブ | スマホ(375px)〜PCまで対応。エディタはモバイルで縦積みレイアウト |
| パフォーマンス | 変換は縮小解像度(最大168セル・3×3サブピクセル解析)で実行し、パラメータ変更はデバウンス付きで再描画 |
| 決定性 | 同じ画像+同じ設定 → 常に同じ刺繍結果（乱数は固定シード） |
| アクセシビリティ | フォームラベル・フォーカスリング・キーボード操作可能なコントロール |
| プライバシー | 画像はブラウザ外に送信しない（全処理クライアントサイド完結） |

## 9. 本プロトタイプの実装範囲と本番化ロードマップ

### 実装済み（このフォルダで完結）
- 全画面・全フロー（閲覧→変換→調整→配置→カート→注文→管理）
- 刺繍変換エンジン（減色・糸マッピング・タタミ/クロス描画・針数計算・SVG書き出し）
- 注文データの永続化（localStorage）と運営用管理画面

### モック（本番では差し替え）
| 項目 | プロトタイプ | 本番 |
|---|---|---|
| 決済 | 入力検証のみ（課金なし） | Stripe / PAY.JP 等 |
| 注文保存 | localStorage | バックエンドDB（Supabase / Firebase 等） |
| メール通知 | なし | 注文確認・発送通知メール |
| 量産用刺繍データ | **実装済み**: PES(刺しゅうPRO対応)・DSTを直接生成 | 実機での試し縫い検証 → 生地別パラメータ（密度・下打ち種類）のチューニング |
| 在庫管理 | なし | 商品・色別在庫 |

### ロードマップ
1. **Phase 1（本プロトタイプ）**: 体験の完成・受注フローの検証・見た目の世界観確立
2. **Phase 2**: バックエンド化（DB・決済・メール・会員/再注文）、独自ドメイン公開
3. **Phase 3**: 刺繍機データ(DST/PES)の自動生成パイプライン、写真向け前処理強化（人物切り抜きAI等）、レビュー・ギャラリー投稿機能

## 10. 運用フロー（受注〜発送）

```
注文受付（#/admin に着信）
 → PES（刺しゅうPROで微調整する場合）または DST をダウンロード
 → ミシンへ転送（USB/無線） → 試し縫い → 本縫い → 検品
 → ステータス「発送済み」に更新 → 発送
※ 仕様書JSONに糸リスト（いとまき品番⇔ブラザー糸番号）と実針数が記載される
```
